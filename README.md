# capa-language-website

The marketing + learn-the-language site for [Capa](https://github.com/nelsonduarte/capa-language).
Served at [`capa-language.com`](https://capa-language.com) from
Cloudflare Workers Static Assets.

## How the site is built

The site is generated by `build.py` from body fragments in `_bodies/`.
A single Python template wraps each fragment with the shared nav,
footer, head metadata and a strict Content-Security-Policy, then
writes the finished pages into `dist/`, which is what Cloudflare
deploys. `dist/` is gitignored and rebuilt on every deploy; the
generated pages are no longer committed. `python build.py --in-place`
writes into the repository root instead, for a local preview from the
repo root.

```
build.py           the generator (template + nav + footer + CSP + sitemap)
_bodies/           per-page body fragments (one file per page)
_additions.css     page-specific CSS appended to the generated stylesheet
fonts/             self-hosted woff2 fonts + OFL licenses + PROVENANCE.txt
capa_logo.svg      vector logo / favicon / OG image
.nojekyll          tells GitHub Pages not to run Jekyll (so _bodies/ and
                   _additions.css, which start with "_", are left alone)
```

Everything below is generated by `build.py` into `dist/` (do not
hand-edit; edit the fragment in `_bodies/` and rebuild):

```
index.html         landing page
why.html           the pitch
compare.html       positioning vs adjacent languages and claim sources
study.html         the capability-recall study
start.html         install + first program
migrating.html     gradual migration from Python
reference.html     language reference
stdlib.html        standard library reference
regulatory.html    CRA / NIS2 / DORA / SSDF / SCVS mapping
agent-demo.html    capability-bound LLM agent showcase
roadmap.html       project roadmap
brand.html         logo + brand guidelines
community.html     where to ask + contribute
privacy.html       privacy policy
learn/             14-chapter tutorial sequence + index
styles.css         single generated stylesheet (fonts, base, additions)
menu.js            mobile menu + accessible Docs dropdown (external, CSP-safe)
sitemap.xml        sitemap (generated from the page list)
robots.txt         crawler rules (generated)
CNAME              capa-language.com (GitHub Pages custom domain; generated
                   only by --in-place, omitted from dist/)
manifest.html      redirect to learn/14-from-source-to-sbom.html (legacy URL)
capa_logo.png      OG image / raster fallback (not generated; kept as-is)
```

### Design and constraints

- Self-hosted fonts under `fonts/` (Inter, Source Serif 4, IBM Plex
  Mono); no third-party requests at all.
- A strict CSP on every page: `default-src 'self'`, no inline
  scripts or styles, no `unsafe-inline`. Former one-off inline
  `style=""` attributes in the fragments are lifted at build time
  into generated classes in `styles.css`.
- Deeper Markdown documents (semantics, packages, regulatory
  mapping, CVE case studies, paper draft) live in the main Capa
  repo's [`docs/`](https://github.com/nelsonduarte/capa-language/tree/main/docs)
  folder and are linked from this site via absolute GitHub URLs.

## Regenerate

```bash
git clone https://github.com/nelsonduarte/capa-language-website
cd capa-language-website
python build.py        # writes the pages + assets into dist/
python -m http.server 8000 --directory dist
# open http://localhost:8000
```

`build.py` recreates `dist/` from scratch on every run, so only the
fragments in `_bodies/` need to be committed.

## Deploy

Cloudflare (Workers Static Assets) builds with `python build.py --dist`
and deploys `dist/` via `npx wrangler deploy`; `wrangler.jsonc` points
`assets.directory` at `dist`. See
[`DEPLOY-CLOUDFLARE.md`](DEPLOY-CLOUDFLARE.md).
